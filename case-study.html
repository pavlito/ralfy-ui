The Problem

Three surfaces, one visual language

Ralfy is a LinkedIn tool: custom feeds, AI comments, reply drafts. It ships as a React web app and a Chrome extension that injects UI directly into LinkedIn pages. Each surface has different CSS constraints — but users expect one consistent look.

Architecture

Token flow

One CSS file defines all values in OKLch. Three mechanisms deliver them to each surface. Change --primary once, it propagates everywhere.

tokens.css (OKLch)
@theme inline → Tailwind → React App
:host CSS vars → Shadow DOM → Comment Generator
JS token object → style={{}} → Feed Manager

Predictable dark mode

Lightness is an independent axis in OKLch. Adjust L without color drift. In HSL, reducing lightness shifts hue — grays turn blue, yellows turn green.

Perceptual uniformity

A 10-step gray ramp in OKLch looks evenly spaced to the eye. In HSL it doesn't. Eliminates the "this shade looks off" feedback loop.

Zero conversion

Tailwind v4's @theme inline maps CSS custom properties to utility classes natively. OKLch values pass through without transformation.

Components

Light & dark, from one set of tokens

Toggle .dark on <html> — everything updates. No conditional rendering, no component code changes. Semantic tokens resolve to the correct values per mode.

AI Workflow

CLAUDE.md turns compliance into a toolchain capability

A machine-readable spec file lists every token class, component API, and variant option. AI coding tools read it before generating code — design system compliance goes from manual review to automated output.

Without CLAUDE.md
// AI generates plausible but non-compliant code
<button
  className="bg-[#1a1a2e] text-white
    px-4 py-2 rounded-md"
>
  Save
</button>

// Hardcoded hex won't update with brand
// No dark mode. No token inheritance.
With CLAUDE.md
// AI reads the spec, generates compliant code
<Button
  variant="primary"
  size="md"
>
  Save
</Button>

// Inherits tokens, dark mode, a11y,
// focus states — for free.

The style guide review step disappears. A new engineer describes what they need; the AI generates system-compliant code.
